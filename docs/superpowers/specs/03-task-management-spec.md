# 任务管理设计文档

## 1. 概述
本文档定义任务输入、分配、跟踪和完成的全流程设计。

## 2. 任务输入

### 2.1 输入组件
```
┌────────────────────────────────────────────────────────┐
│  请描述您需要完成的任务...                               │
│                                                        │
└────────────────────────────────────────────────────────┘
                                           [ 提交任务 ]  │
```

### 2.2 输入验证
| 验证项 | 条件 | 错误提示 |
|--------|------|----------|
| 非空 | trimmed length > 0 | "请输入任务描述" |
| 长度 | length <= 1000 | "任务描述过长，请控制在1000字以内" |
| 格式 | 无特殊限制 | - |

### 2.3 提交状态
- **禁用**: 输入为空或提交中
- **加载中**: 显示 spinner，文字变为 "提交中..."
- **成功**: 清空输入框，Toast "任务已提交"

## 3. 任务分解流程 (Planner)

### 3.1 分解过程
```
用户提交任务
       │
       ▼
┌──────────────┐
│ Planner 接收 │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 分析任务需求  │  ← 日志: "正在分析任务..."
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 分解为子任务 │  ← 日志: "分解为 N 个子任务"
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 分配给       │
│ Executor     │  ← 日志: "任务已分配给 Executor"
└──────────────┘
```

### 3.2 分解结果展示
```
任务分解结果:
├── 1. 分析需求并设计算法
├── 2. 实现排序算法代码
├── 3. 编写单元测试
└── 4. 优化性能

[ 开始执行 ]  [ 取消任务 ]
```

## 4. 任务执行流程 (Executor)

### 4.1 执行过程
```
接收子任务
    │
    ▼
调用 Codex API
    │
    ├── 成功 ──► 生成结果
    │               │
    │               ▼
    │         发送给 Reviewer
    │
    └── 失败 ──► 记录错误，重试 (最多3次)
                    │
                    ▼
              标记任务失败
```

### 4.2 执行状态
- **pending**: 等待执行
- **in_progress**: 执行中
- **completed**: 执行成功
- **failed**: 执行失败

## 5. 审查流程 (Reviewer)

### 5.1 审查过程
```
接收 Executor 结果
        │
        ▼
验证结果正确性
        │
        ├── 通过 ──► 标记任务完成
        │
        └── 不通过 ──► 返回 Executor 重新执行
                           (最多2次循环)
```

### 5.2 审查标准
- 结果是否解决了原始任务
- 代码是否有明显错误
- 是否符合基本质量要求

## 6. 任务列表

### 6.1 列表视图
```
┌─────────────────────────────────────────────────────┐
│ 任务列表                    [全部 ▼] [最新排序 ▼]   │
├─────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────┐ │
│ │ 🔵 进行中  帮我写一个排序算法                     │ │
│ │        Planner → Executor → Reviewer             │ │
│ │        10:00 创建  │  预计剩余: 2分钟            │ │
│ └─────────────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────────────┐ │
│ │ 🟢 已完成  帮我写一个求和函数                     │ │
│ │        Planner → Executor → Reviewer             │ │
│ │        09:30 创建  │  09:35 完成                 │ │
│ └─────────────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────────────┐ │
│ │ 🔴 失败    帮我连接数据库                        │ │
│ │        Planner → Executor                        │ │
│ │        09:00 创建  │  09:01 失败: 连接超时       │ │
│ └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 6.2 筛选条件
- 全部
- 进行中
- 已完成
- 失败

### 6.3 排序方式
- 最新创建
- 最近更新
- 优先级

│  ┌──────────────────────────────────┐  │

## 8. 任务状态持久化
- 任务数据存储在后端内存/数据库
- 支持任务 ID 查询
- 历史任务保留 7 天
