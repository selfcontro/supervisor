# Agent 交互设计文档

## 1. 概述
本文档定义用户与 Agent 节点之间的交互行为和界面元素。

## 2. 交互流程

### 2.1 查看 Agent 详情
```
用户点击节点 → 右侧滑出详情面板 → 显示 Agent 信息 → 用户可执行操作
```

### 2.2 手动触发 Agent 操作
```
用户点击操作按钮 → 确认对话框 → 执行操作 → 更新状态 → Toast 反馈
```

### 3.2 面板宽度
- 桌面端: 400px
- 移动端: 100% 全屏

### 3.3 动画
- 滑入动画: 300ms ease-out
- 关闭动画: 200ms ease-in

## 4. 操作按钮行为

### 4.1 重试按钮
- **条件**: Agent 状态为 error
- **行为**: 重新执行最近失败的任务
- **确认**: 需要确认对话框
- **反馈**: Toast "任务已重新提交"

### 4.2 暂停按钮
- **条件**: Agent 状态为 working
- **行为**: 暂停当前任务执行
- **确认**: 无需确认
- **反馈**: Toast "任务已暂停"

### 4.3 历史按钮
- **条件**: 无
- **行为**: 打开任务历史列表弹窗
- **弹窗内容**: 最近 20 个任务的列表

## 5. 状态流转

### 5.1 Agent 状态机
```
        ┌──────────────────────────────────────┐
        │                                      │
        ▼                                      │
     ┌─────┐     执行完成      ┌──────────┐    │
────►│idle │─────────────────►│completed │────┘
     └─────┘                   └──────────┘
        │                          ▲
        │ 接收任务                  │
        │                          │
        ▼                          │
     ┌───────┐     执行完成        │
────►│working │──────────────────►│
     └───────┘                     │
        │                          │
        │ 发生错误                  │
        ▼                          │
     ┌─────┐                       │
     │error│───────────────────────┘
     └─────┘     重试成功
```

### 5.2 状态显示
| 状态 | 显示文字 | 颜色徽章 | 可执行操作 |
|------|----------|----------|------------|
| idle | 空闲 | 绿色 | 分配任务 |
| working | 工作中 | 黄色 | 暂停 |
| completed | 完成 | 蓝色 | 查看结果 |
| error | 错误 | 红色 | 重试 |

## 6. 实时更新

### 6.1 WebSocket 事件
- `agent_status`: Agent 状态变化
- `task_update`: 任务更新
- `log_entry`: 新日志条目

### 6.2 更新机制
- 状态栏显示 "实时同步中" / "同步断开"
- 断开时显示重连按钮
- 自动重连间隔: 2s, 4s, 8s, 16s, 32s (最多5次)

## 7. 触控适配
- 面板支持手势滑出/滑入
- 长按节点显示快捷操作菜单
- 支持 3D Touch / Haptic Feedback
